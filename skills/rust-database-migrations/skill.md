# Rust Database Migrations

## When to Activate
- Setting up database migrations for a Rust project
- Writing new migrations
- Managing schema changes
- Configuring CI for migrations

---

## SQLx Migrations

```bash
# Install sqlx CLI
cargo install sqlx-cli --features native-tls,postgres

# Create a migration
sqlx migrate add create_users_table

# Run migrations
sqlx migrate run

# Revert last migration
sqlx migrate revert

# Check status
sqlx migrate info
```

### Migration File Structure
```
migrations/
├── 20240101000001_create_users_table.sql
└── 20240102000000_add_user_indexes.sql
```

```sql
-- 20240101000001_create_users_table.sql
CREATE TABLE IF NOT EXISTS users (
    id BIGSERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ
);

-- Down migration (for revert)
-- DROP TABLE IF EXISTS users;
```

### Embedding Migrations in Binary
```rust
// src/main.rs
sqlx::migrate!("./migrations")
    .run(&pool)
    .await
    .expect("Failed to run migrations");
```

## Diesel Migrations

```bash
# Setup Diesel
cargo install diesel_cli --features postgres

diesel setup
diesel migration generate create_users
diesel migration run
diesel migration redo  # rollback + re-run
```

### Diesel Schema
```rust
// src/schema.rs (auto-generated by Diesel)
table! {
    users (id) {
        id -> BigInt,
        email -> Varchar,
        name -> Varchar,
        created_at -> Timestamptz,
    }
}
```

---

## Migration Best Practices
- Never modify existing migration files after they're applied
- Always write down migrations for reversible changes
- Test migrations against a copy of production data before deploying
- Include migrations in CI pipeline
- Version control all migration files
